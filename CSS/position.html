Position en CSS

En esta clase veremos como se comportan las instrucciones position y display, siendo estas parte fundamental para el desarrollo de páginas web modernas y bien estructuradas.

Como ya hemos visto en el curso de Uniwebsidad, con el uso de display y position es posible realizar paginas web con diseños modernos.

Instrucción position

La funcion de la regla position es la de posicionar los elementos y darles propiedades para que estos interactuen de maneras diferentes con las cajas aledañas, afectando el flujo de HTML.

Al momento de introducir las propiedades position tenemos la posibilidad de usar 4 propiedades nuevas: top, rigth, botton,left. Estas propiedades nos permiten modificar la posición donde se mostraran las cajas en cuestión.
Hay que tomar en cuenta que al momento de que hayan 2 instrucciones del mismo eje (top-bottom, left-right), los valores que CSS va a utilizar al ser interpretados como mas importantes van a ser top y left. Al momento de que tengamos 2 instrucciones para el mismo eje EJ: "top:20px; bottom:30px;" CSS va a utilizar el valor de mayor relevancia que en este caso seria top.

z-index

Con la propiedad z-index podemos darle un orden a los elementos que se encuentran solapados siendo el valor más alto el que se encuentre más cerca de nuestro campo de visibilidad.

z-index: 1;
z-index: 2;
z-index: 3;

En el momento de que no se indique ningun z-index, CSS determina que sera visible jerarquicamente aquellos elementos que se ejecuten de ultimo en el codigo.

Tipos de position

Existen 4 tipos diferentes de position los cuales veremos a continuación:

static

La posicion static es la que viene por defecto en HTML. Esta simplemente posiciona el elemento uno al lado del otro tomando en cuenta si es en linea(al lado) o si es en bloque(debajo).

relative

Al tu darle valor relative en la propiedad position, el elemento que contenga este valor va a ocupar el mismo espacio en el flujo de HTML no permitiendo a los demas elementos ocupar ese espacio. Pero si modificamos sus valores de posisición (top,right,bottom,left) este se movera sin permitir que los demas elementos ocupen su lugar.

absolute

En el caso del position absolute hace que la caja en cuestion se superponga a las demás cajas con position relative y esta no ocupa espacio dentro de el flujo de HTML. Este elemento desde la izquierda del todo como valor inicial.

Como se puede observar en este ejemplo la caja 3 y la 6 se separan del resto del flujo de html, pero respetando el espacio comprendido entre estas y los demás elementos en bloque hermanos.

Con position absolute, si no definimos la altura (heigth) y la anchura (width) este se auto ajusta al contenido de la caja.

La otra caracteristica intersante es que con el position absolute toma como posicion inicial al la esquina superior izquierda del elemento contenedor siempre y cuando este este posicionado. Si el elemento contenedor no esta posicionado los elementos con la posición absolute toman como posicion inicial las esquinas superior izquierda del view port(de la página en si o de la etiqueta body que es la que contiene a todos los demas elementos)

fixed

Con fixed podemos inplementar cajas que siempre se mantengan en una misma posición en el view port manteniendose estaticamente allí y superponiendose al resto de elementos del documento HTML.

Este elemento respeta los z-index, por lo tanto cualquier elemento que contenga un z-index superior al de la caja fixed se superpondra sobra esta.

Esta caja es muy util para crear los menús estaticos como los que tienen páginas como facebook, netflix, etc...

sticky

El valor sticky funciona parecido al valor fixed con la diferencia de que este no ocupara el lugar establecido por los parametros (top, bottom) hasta que estos no aparezcan en el port view del usuario. Los Titulos de esta pagina se acomodaran utilizando elementos con la position sticky
