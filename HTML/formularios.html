Formularios en HTML

Veremos los distintos tipos de formularios que nos ofrece html.

hmtl nos brinda herramientas para que nosotros podamos interactuar desde la pagina hacia un servidor para procesar, administrar o almacenar informacion. Para llevar a cabo estas tareas html nos brinda de herramientas como botones, textefields, campos para contraseñas, comprobadores de formato mail, etc... Y con esto podemos realizar paginas mas complejas ya sea para crear buscadores, regisrtros de clientes, administracion de las paginas etc...

En esta clase veremos la manera de crear aplicaciones web sencillas mediante el uso de HTML. (OJO. ESTO SOLO APLICA PARA LA PARTE DE LA ESTRUCTURACION DE LOS DISTINTOS COMPONENTES DENTRO DE HTML. Para usar los distintos componentes que nos ofrece html hay que utilizar la etiqueta <form> para utilizar los formularios. Los principales etiquetas para el desarrollo de aplicaciones web son los siguientes:

  • action = "url" - Indica la URL que se encarga de procesar los datos del formulario.

  • method = "POST o GET" - Método HTTP empleado al enviar el formulario.

  • enctype = "application/x-www-form-urlencoded o multipart/form-data" - Tipo de codificación empleada al enviar el formulario al servidor (sólo se indica de forma explícita en los formularios que permiten adjuntar archivos)

  • accept = "tipo_de_contenido" - Lista separada por comas de todos los tipos de archivos aceptados por el servidor (sólo para los formularios que permiten adjuntar archivos)

  • Otros: accept-charset, onsubmit, onreset

Para declarar que tipo de formulario vamos a utilizar usaremos dentro de la etiqueta <form> la etiqueta <input> que es la que realmente nos proporciona los elementos que vamos a utilizar para desarrollar las aplicaiones web. Esta Consta de los siguientes atributos:

  • type = "text | password | checkbox | radio | submit | reset | file | hidden | image | button" - Indica el tipo de control que se incluye en el formulario

  • name = "texto" - Asigna un nombre al control (es imprescindible para que el servidor pueda procesar el formulario)

  • value = "texto" - Valor inicial del control

  • size = "unidad_de_medida" - Tamaño inicial del control (para los campos de texto y de password se refiere al número de caracteres, en el resto de controles se refiere a su tamaño en píxel)

  • maxlength = "numero" - Máximo número de caracteres para los controles de texto y de password

  • checked = "checked" - Para los controles checkbox y radiobutton permite indicar qué opción aparece preseleccionada

  • disabled = "disabled" - El control aparece deshabilitado y su valor no se envía al servidor junto con el resto de datos

  • readonly = "readonly" - El contenido del control no se puede modificar

  • src = "url" - Para el control que permite crear botones con imágenes, indica la URL de la imagen que se emplea como botón de formulario

  • alt = "texto" - Descripción del control

Diferentes Elementos de la etiqueta <input>

  • text

    Este elemento se implementa para enviar datos de caracteres o numericos al servidor.

    Como se puede observar. Este permite insertar texto y muestra los caracteres tal cual se escriben.

  • Password

    Este elemento es parecido a text pero se diferencian en que este no muestra los caracteres que estamos escribiendo.

  • Checkbox

    El checkbox es un elemento que nos permite seleccionar casillas para que el servidor luego los interprete. Pueden haber tantas como el programador desee.

  • radio

    El elemento radio funciona al igual que el elemento checkbox pero con la diferencia de que solo puede haber seleccionado uno a la vez. Al momento de seleccionar otro radiobutton el que este seleccionado con anterioridad se deseleccionara. Se recomienda usar el atributo checked para aquellos formularios en los que se deba seleccionar un elemento obligatoriamente (lo mismo para el elemento checkbox)

  • Submit

    El elemento submit es un boton que permite enviar la informacion y la eleccion del usuario al servidor. De esta manera este es el encargado recopilar y enviar los elementos al servidor en cuestion.

  • Reset

    Este boton sirve para borrar todos los datos que el usuario haya modificado con anterioridad. Este es el tipico boton que se suele usar para las opciones de "Empezar de 0" o "Borrar todo" etc... Este es muy util para emplear en caso de querer realizar formularios con gran extension de datos.

  • File

    Este Elemento es muy util ya que permite navegar entre tus archivos para buscar alguno en especifico mediante el "tree" de el discoduro. Muy util para cuando se necesita enviar archivos enteros al servidor. Adjunta archivos para luego enviarlos al servidor.

  • hidden

    Este elemento permite enviar datos al servidor pero sin que el usuario pueda visualizar en ningun momento que se hace con este. No muestra ningun o elemento en pantalla.

  • image

    Mas facil de recordar como boton de imagen. Este sirve para insertar una imagen que funcione como un boton. (alternativa estetica al tipico "button")

  • Boton

    Este funciona como cualquier boton que hayamos conocido con anterioridad. no hace falta explicar mas.
