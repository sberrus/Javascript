Manejo Control de Flujo en JavaScript

Control de Flujo

Se refiere a control de flujo a el orden en el que se ejecutan los programas dentro del DOM. Con el uso de llaves "{}" podemos crear ambitos dentro del código js.

Ambitos dentro de JS

Se refiere a Ambito a los bloques internos con sus respectivas prioridades que permite ejecutar en cierto orden las distintas líneas de código. (Un poco lo que se hace al momento de hacer funciones. Que hasta que no se ejecute la función, no se ejecuta el código que esta dentro de está).

El ámbito permite que el código se comporte de cierta manera y esto junto con el scope nos permite cambiar el comportamiento del código en sí.

Sentencias de Control de Flujo

Estas sentencias son las que nos permiten modficar el flujo del código. De esta manera podemos indicar si un código se debe ejecutar o no. Estas sentencias son las que ya conocemos cono if() switch() etc...

Uso de la estructura Switch (case, default, break)

Igual manejo, uso y sintáxis que en Java.

Sentencias de manejo de Excepciones

Si por cualquier razón deseamos trabajar con los errores que se pueden generara dentro del programa por cualquier razon podemos hacer uso del manejo de excepciones en JS.

Uso de las Excepciones

Las Excepciones sirven para que en el momento de que querramos hacer algo que pueda generar algún error en el programa, este pueda seguir funcionando sin tomar en cuenta dicho error. Hace que continue con el flujo normal del programa sin que tenga en cuenta dicho error. Existen 3 tipos de excepciones, las ECMAScript, las DOMError y las DOMException.

try...catch...finally

try..catch como en java, es un bloque de código que nos permite manejar los erorres que nos devuleve el programa y poder manejarlos de cierta manera como deseemos o que en caso de que ocurra, el programa lo omita haciendo que mejore el funcionamiento del mismo.

Bloque try

En este bloque irá la sentencía que se intentará ejecutar y donde pudierá generarse cualquier error. El try no puede ejecutarse si no esta el catch de ese bloque. Por lo tanto van siempre de la mano uno del otro.

Bloque catch(error)

En este bloque especificamos que se debe hacer con el error que devuelve este bloque de código. El parámetro que recibe este bloque es el que nos permite realizar las acciones correspondientes en el momento de que haya algun error.

Por definición el catch nos devuelve un objeto como parámetro que podemos, o no, usarlo.

Cosas a tomar en cuenta la trabajar con excepciones

Las excepciones no manejan todos los tipos de errores que pudiera contener el código. Hay expresiones o errores (por ejemplo los de sintaxis) que no nos permiten trabajarlos con el uso de estos métodos.

typeof

Este método es utilizado para saber de que tipo es la variable a la que se la aplicamos. Devuelve booleano, object, number etc...

Finally

Este bloque de código, si lo declaramos, se ejecuta siempre al final. Indiferentemente si hay algún error o no. Este bloque de código no es obligatorio para que el manejo de excepciones sea efectivo.

Otra de las características principales de finally es que el valor que retornemos mediante este tiene prioridad inclusive más que el try.

Cosas a tomar en cuenta

Si es verda que el uso del manejo de excepciones es una herramienta que ayuda mucho en las labores de desarrollo de software, no se debe utilizar para todo. Principalmente se debe priorizar el debuggear el código de manera nativa.

En general no se debe usar el try..catch generalizadamente. Esto se usa en mayor medida con los async y los await
