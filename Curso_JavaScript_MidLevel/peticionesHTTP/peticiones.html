Peticiones HTTP en JavaScript

¿Qué es una petición HTTP?

Se refiere a petición HTTP a la capacidad que tiene un ordenador de pedirle información o cualquier cosa a un servidor y que este nos devuelva algo, per no solo esto, nosotros también podemos enviarle cosas al servidor y que este nos devuelva cosas con las que podamos trabajar. En general se tiende a enviar datos en formato JSON.

¿Qué es un JSON?

Un archivo de formato JSON, es un archivo que almacena la información de manera similar a como lo hace un objeto en JavaScript. De por si JSON significa JavaScript Object Notation. Su estructura es exactamente igual a un array asociativo. objeto = [{"nombre":"Samuel"},{"nombre":"Pedro"},...].

La unica diferencia es que el JSON la variable asociativa se tiene que encerrar entre comillas dobles. Si las variables no se cierran entre comillas dobles es un array asociativo.

JSON Serializado

Se refiere a JSON serializado a un JSON en formato de cadena de texto (encerrado entre comillas, comillas dobles, etc...).

Se usan los JSON serializados ya que de esta manera los demás programas y lenguajes de programación pueden recibir esta cadena de texto y parsearla para poder utilizar los datos que esta contiene.

stringify()

Para serializar un objeto para convertirlo a JSON de manera que pueda ser enviado para luego ser interpretado, se hace uso del método stringify(objeto). Este método nos devuelve un objeto JSON serializado (en cadena de texto).

parse()

Si recivimos desde otro sitio algun JSON serializado para poder acceder a los valores dentro de este hay que parsearlo con el metodo JSON.parse(JSON). Después de haberlo parseado y alojado en algún sitio, podemos hacer uso de sus valores.
